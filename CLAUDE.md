# CLAUDE.md

Guidance for Claude Code (claude.ai/code) working in this repository.

## What this is

**Crash Bet** — a betting game about reading physics, built on a deterministic crash sim and a seeded procedural low-poly world. Every round deals a generated traffic scene ten seconds from disaster, the player bets on what the chaos will do, and the sim settles every bet with zero ambiguity. Vehicles and property only — no people, ever.

Vanilla JS + Three.js r169 + Rapier 3D 0.19, both vendored (`libs/`, import map in `index.html`). No build step, no package.json, no framework, fully offline, installs as a PWA. The game is feature-complete and not deployed.

**App shell**: boots to a main menu gated behind a boot preloader (physics + Rapier wasm + fx pools + showroom warm up behind a progress bar; `preloaded` flips at 100 %). Menu (G6: one glass card — primary Continue CTA + a 3×2 tile grid with inline SVG icons): **Continue/Resume** (campaign) · **Daily** (one date-derived scene a day) · **Stats** (record + 16 achievements) · **Seed** (Exhibition) · **Garage** (showroom of all 280 models) · **How to** · **Crash test** (8 seeded set-piece wrecks) · **New run**. The menu/garage backdrop env rotates daily through the good-looking presets (`MENU_ENVS`). In game: hamburger → pause menu (Resume / Settings: quality · camera · volume · motion / Fullscreen / Leave) — **no environment picker**: the scene owns its env (the director deals it), and a Settings override was both a spoiler surface and a mismatch with what the odds were priced on. The in-round scene tag shows PLACE + WEATHER only — never the incident name or difficulty (the scene is the mystery; the tell is revealed on the settle card). Dev URL params: `?scene=<seed>~<d>`, `?crash=N`, `?play=1`, `?smoke=1`, `?simtest=1`, `?sheet=1` (vehicles) / `?sheet=scenery` (all scenery models).

**PWA / service worker.** `sw.js` is network-first with cache fallback. Two rules: **bump `CACHE`** (`crash-bet-vN`) whenever any cached asset changes — the old cache is deleted on activate, and a version bump is what stops a returning user pairing new JS with a stale cached `index.html` (that mismatch throws in `mountBetUI` and bricks boot at "warming up"); and **`ASSETS` must name every imported module**, or offline load fails on whichever one is missing. New-element access in `betui.js` that runs at mount time is guarded (`setHidden`, `?.`) so even a transient update skew degrades instead of bricking.

**The editor is mothballed, not deleted**: `js/editor.js` (spawn/gizmo/inspector/URL codec/worldgen UI) is intact but not imported, and its DOM was removed from `index.html` — re-wiring needs that DOM back plus `initEditor(ctx)`. Editor-era URL params (`car`/`prop`/`road`) are no longer parsed. Its UI CSS was removed from `css/style.css`; a few editor-era rules the game shell shares by class name (`.primary`, `#toast`, the touch `input[type=range]` sizing) are kept and marked.

## Running & testing

- Serve over HTTP (ES modules will not load from `file://`): `run.bat`, or `npx http-server -p 5183 -c-1 .`. `.claude/launch.json` has a `garage` config for the browser preview; it uses `autoPort`, so read the port from the preview result rather than hardcoding 5183. **The service worker can serve stale assets during dev** — if an edit does not show up, unregister the SW + clear caches and hard-reload.
- **Smoke test**: `?smoke=1` — builds every registry type × 4 seeds, logs `SMOKE DONE: N/N ok`. Run after changing `vehicles.js`, `families.js`, `parts.js`, `scenery.js` or `lib.js`. It is a coverage count, not a pin — it moves whenever the registries grow (currently 92 vehicles + 184 scenery, × 4).
- **Contact sheets**: `?sheet=1` tiles every vehicle, `?sheet=scenery` every scenery model, into one canvas at `window.__sheet`. Both run synchronously at boot so they work with rAF suspended. Reviewing a batch on the sheet is the only practical way to catch backwards winding, floating parts and wrong-axis rotations — none of those fail the smoke test.
- **Game sheets (P4)**: `?sheet=scenes` deals one seed per topology through the REAL `startScene` at d 1/5/10 (recorder, env pool, terrain, veg, wetness, merge/freeze — nothing mocked), fast-forwards to the incident tick with the hooks nulled and tiles the wide fit; `?sheet=weather` renders all 11 envs × 11 kinds over a fixed two-car vignette via `rollWeather`'s `forceKind` (dev-only third arg — the pick still happens so no stream shifts; off-distribution cells like a desert blizzard render on the QA sheet even though `WEIGHTS` never deals them). Both are async (they need the boot preload), land per-cell renderer stats on `window.__sheetStats` (draw calls/tris/geos/heap — a free 66-round soak), and run on `Econ.memoryStore()` so a QA sweep can never move the player's profile. Known artifacts of the harness, not the game: precip renders as a clump at the scene centre (the camera-following volume parked at spawn — weather is frozen at capture), and each topology row shows ONE env/weather (both derive from the seed alone). This caught the water-foam lattice and the black canyon floor; the model sheets could not have.
- **Determinism self-test**: `?simtest=1` — runs every `TEST_SCENARIOS` entry in physics.js twice (300 fixed steps), FNV-hashes every body transform per step (debris included) plus the final crumpled geometry. Run after changing `physics.js`, `deform.js`, `props.js`, `scenery.js`, `roads.js`, `worldgen.js` or `director.js`. Hashes are stable across node and browser (same wasm f32 math).
- **Pin gate**: `node tests/simpins.mjs` — `?simtest=1` only proves run-to-run stability; this proves the values never MOVED, which is the invariant that actually protects the sim. Pins (in the test file) cover legacy · extended · scenery · roads · deck · bridge · water · waterv2 · carnage · director · wxgrip · worldgen, each as a tick-transform hash plus a final geometry hash. **None may drift unless physics/deform intentionally changed** — update the pinned value and record why next to it when they do. A pinned scenario that bypasses the generator does not prove the generator works, so `generateScene` (and the pins themselves) must SPREAD `topo.world`, never re-list its keys, or a world key is silently dropped.
- **Scrub gate**: `node tests/scrub.mjs` — the freeze scrub rewinds by re-simming from t0, so this asserts a scrub round-trip lands on the EXACT state a clean run reaches (per seed × difficulty, rewinding to arbitrary ticks including 0). A money-correctness gate: the slip settles against a recording whose incident starts at `INCIDENT_TICK`, so a sim resuming from a different state plays a scene the odds were never priced on.
- **Economy suite**: `node tests/economy.mjs` — payout flooring, bankroll floors, once-per-seed ledger, parlay caps, mid-round resume, the Exhibition rule, the daily seed (shared across profiles, one paying attempt, streak arithmetic across month/year boundaries), achievements (idempotent, Exhibition unlocks nothing, a throwing test cannot break settlement), plus a real scene → markets → settlement integration pass.
- **Weather gate**: `node tests/weather.mjs` — weather never touches the sim, so no pin can catch a mistake in it. It guards the silent-fallback class: `WEIGHTS` falls back to `proving` on an unknown env id, so it asserts `ENVS` and `WEATHER_ENVS` agree **in both directions**. Plus descriptor invariants, roll determinism, grid staying clear, ≥4 kinds per env, and the grip block (`gripFor` returns 1 for anything it does not recognise — dry kinds must be exactly 1, and grip falls monotonically along the rain ramp). Run after touching `weather.js` or `ENVS`.
- **Weather neutrality gate**: `node tests/wxneutral.mjs` (P4) — proves the LOOK can never touch the MONEY from the outside: an alien descriptor (storm/soaked/gale) with `grip` carried over bit-for-bit must produce an identical `rec.hash`; stripping `world.weather` on a no-grip scene must too; and grip 0.70 → 1 MUST move the hash (the teeth — a refactor that quietly stops grip reaching the tyres makes the neutrality half pass vacuously). The sim reads exactly one weather field (`wx.grip`, physics.js) and this pins that contract.
- **Scene sweep**: `node tests/sweep.mjs [seedsPerD]` — deals across d 1/5/10, records to rest, reports coverage per topology × template. Hard defects: a car↔car hit before the incident tick, no markets, a missing headline pair. Quality *budgets* (rates, not per-scene defects): ran-to-cap and eventless are tracked with headroom — both are legitimate outcomes. The pre-incident check counts **car↔car only** (the recorder logs kerb and ground contacts as `hit` too, and clipping a kerb on the approach is not a choreography failure).
- **Monte Carlo gate**: `node tests/montecarlo.mjs [roundsPerD]` (~9 min at default) — bets every market to-win-$100 and asserts the realized house edge sits inside the per-difficulty band around `marginFor(d)`. Seeds are fixed, so re-running the same size is deterministic and cannot tell you whether a band miss is noise — vary `roundsPerD` to resample. Calibration seeds (`cal…`) and gate seeds (`mc…`) are disjoint, so the measurement is out-of-sample. Beware short runs: at low counts the estimator's σ is large, so a small sample "fails" on noise alone.
- **Odds calibration**: `node tools/calibrate.mjs [n]` prints the empirical CALIB table to paste into `js/markets.js`. Regenerate whenever director/physics changes move outcome statistics. It **MUST pass `{ all: true }`** — offering is itself a function of CALIB, so measuring rates over only the offered set feeds a truncated sample back as the full-population prior and does not converge. It also prints observed/expected per difficulty band with a 95 % interval — the number the Monte Carlo actually tests (edge ≈ `1 − (1 − margin)·O/E`).
- **Headless physics in node** (much faster than the browser loop): `node_modules/three/` is a gitignored shim so the whole chain imports in node. Import via `pathToFileURL`, build a `CrashSim`, call `stepOnce()`, assert on `body.translation()` / `hashState()`.
- Syntax check without a browser: copy a module to `.mjs` and run `node --check`.
- **Headless visual verification**: `window.__app` exposes `{ renderer, scene, camera, controls, REG, env, fitCamera, invalidate, startGame, leaveGame, setCamMode, fly, flyUpdate, showroom, startCrash, nextCrash, replayCrash, crash, crashFx, startScene, resumeRound, seekPreview, round, preloaded, pump, weather, rollWeather, veg }`. **`__app.pump(now)` runs one real frame** (camera tweens, sim, fx, HUD, render) with a synthetic clock — loop it with `now += 1000/60` to drive the game headlessly instead of re-implementing the loop. DOM overlays can be rasterized via SVG `foreignObject` with the stylesheet inlined (serialize with `XMLSerializer` — raw `innerHTML` is not valid XML).

## Architecture

Module chain (each imports the previous): `lib.js` → `parts.js` → `families.js` → `vehicles.js` → `main.js` (+ `names.js`). Sim: `deform.js` + `props.js` → `physics.js` → `main.js` (+ `env.js` + `fx.js`); `terrain.js` is pure and imported by `env.js` for the mesh and by `physics.js` for the drivable heightfield — both must sample the SAME `makeHeightField` or collider and landscape drift apart; `water.js` imports `lib.js` and is owned entirely by `env.js` (env holds the lifecycle, water.js the look); `weather.js` imports only `lib.js` and is consumed by `main.js` **and by `director.js`**, which calls its two pure functions (`rollWeather`/`gripFor`) to put the scene's weather on the scenario — env.js still takes a weather *descriptor*, never the module; `vegetation.js` imports `lib.js` + `scenery.js` and is driven by `main.js` from `env.terrainField`. Game: `director.js` (imports roads + worldgen + weather's pure roll, NEVER physics) → `recorder.js` (imports physics + director + roads) → `main.js`; physics.js imports director.js only for its pinned `director` scenario. `signals.js` is pure (program + tick evaluation, no THREE, no rng at eval time) and is imported by `director.js` to generate, `physics.js` to obey and `main.js` to light the lamps. `markets.js` → `economy.js` → `betui.js`. `achievements.js` is pure and imports nothing. `editor.js` is dormant. Rapier is lazy-loaded (2.2 MB) during the boot preloader.

**Physics (`physics.js`) — determinism is the product.** Fixed 60 Hz timestep + accumulator; rendering interpolates and never feeds back. Zero `Math.random()` in sim code; world creation order is the backbone: ground → arena walls → roads → props → cars (array order). Chassis colliders are convex hulls from each slab's `userData.pt`, rescaled to a per-category target mass (`CAT_PHYS`) with a no-contact ballast collider lowering the COM. Wheels come from `userData.wheel` tags; duals dedupe, inline bikes get invisible outriggers, tracked vehicles get 4 virtual corner wheels. Raycast vehicle controller per car. Per-car tuning multipliers default to exactly 1.0 (`rest: null` → 0.12) so legacy scenarios hash bit-identically. Scenario shape `{ world: {gravity, arena, walls, water, terrain, weather}, roads, junctions, props, cars }`; "perfect reset" = rebuild from the scenario (~30 ms for 10 cars). **Incremental editing** (`replaceCar`/`appendCar`/…) touches one object for paused responsiveness and does NOT promise a deterministic world — that is why the editor flags `dirty` and Play/Reset/Step always start from a full `build()`. **Damage model**: `CAT_PHYS` carries per-category `sus`, `aDamp`, `crumpleK`, `wheelTough`; impacts charge nearby wheels, past `WHEEL_BENT_AT` a wheel steers crooked, past `WHEEL_DETACH_AT` it tears off into a real dynamic debris body inheriting `v + ω×r` (`hashState` folds debris in; any new debris path MUST register with `this.debris` or cleanup and hashing miss it). Six render-side hooks — `onImpact/onScrape/onGlass/onDetach/onSplash/onSunk` — fire inside `stepOnce`; **the only sanctioned write-back is `sim.speed`** (wall-clock pacing).

**Deformation (`deform.js`).** Non-indexed geometry, vertices weld by position (half-mm quantization), displacement applies per weld group so faces never tear. Depth scales with contact Δv, radial falloff + position-hash grain, accumulated crush clamped per group. Visual only — colliders stay rigid. Per-group zone stiffness (nose/tail crumple deep, cabin resists, roof gives in rollovers), work hardening, ridged crease noise + tangential shear, and two-stage glass (crack → shatter) driven by direct impact overlap plus inverse-square "frame shock" that pops windshields from bumper hits.

**Roads (`roads.js`).** Catmull-Rom spline roads (centripetal, so tight zig-zags never cusp). `buildRoad(spec)` sweeps asphalt + geometry lane markings, curbs, optional sidewalks and crosswalks, returning `{ group, shapes }` — shapes are curb box-collider recipes physics attaches to ONE fixed body at the origin (the group is never transformed). Zero randomness: segment counts derive from arc length. Strip winding rule: horizontals face up when `o0<o1`; the mirrored side negates offsets AND flips winding — pass offsets ascending for s=+1 or faces render invisible from above. Style bitfield: 1 = double yellow, 2 = sidewalks, 4 = crosswalks, 8 = bridge deck, 16 = rural verge, 32 = swept guardrail (both ignored on a deck).
- **A verge REPLACES the kerb and emits NO collider** (`16`): a gravel shoulder falling to −0.05, so leaving the road is a run-off rather than a kerb strike. `32` sweeps a real guardrail (posts on arc-length stations, beam, colliders at `GRAIL_H`) — the existing `guardrail` scenery kind is a discrete 4 m object, this is the continuous run a mountain road needs. Verge+guardrail together gives the guardrail's colliders.
- **Lane count from width**: past 10.5 m a road gets interior dashes. Visual only — director lane extraction is untouched.
- **Tyre polish** down each wheel path (always on, geometry-derived) is what stops a long straight reading as one flat swatch at grazing angles.
- **Elevation** is opt-in and defaults to exactly 0 (every height is an offset ABOVE the frame's surface, `f.y + y0`, so `+0` is a bit-level no-op — do not "simplify" to absolute heights). Tangents normalise by HORIZONTAL length so the ribbon never narrows as it climbs. Curb colliders keep the yaw-only quaternion on flat roads and only pay for the pitched basis when `isElevated(spec)`. Elevated runs also emit a driving-surface slab (opt-in on `elev`) whose longitudinal overlap is deliberately smaller than the curbs' — a generous overlap leaves the next pitched slab's leading edge proud, a lip that trips a wheel.
- **Substructure** (edge girders, pier bents on ~17 m stations, pier caps, abutments) is drawn when `deck && elev` and is **visual only — nothing reaches `shapes`**. Columns run to `PIER_FOOT` well below 0, so on land the ground buries the footing and over a punched water basin the same column founds itself in the bed — one number, both cases, and roads.js never learns what water is. Girders sweep the full length, buried at grade by the same trick.
**Junctions (`buildJunction`).** `{ x, z, arms: [{a, w}], reach, fillet, style }` → `{ group, shapes }` where **`shapes` is always empty** — a junction is flat asphalt on the y-0 plane, the ground slab already holds cars up (so adding them moves no pin). They live on `scenario.junctions`, are built in `physics.js` between roads and props to share the scene root, and main.js must include them in **both** the wetness pass and the merge/freeze pass. Style bits: 1 stop bars · 2 crosswalks · 4 keep-clear box · 8 turn arrows.
- Everything is drawn **2 mm BELOW** the equivalent road layer, so a stub may overlap the junction freely without z-fighting — the polygon never has to be tangent to anything and arms can be any width at any angle (the trick the old `asphalt_patch` prop used; its mistake was being a fixed square rather than a shape derived from the arms).
- Corner fillets round a REFLEX corner, so they ADD asphalt: centre solves `(X−P)·a = (X−P)·b = r`, both tangent points slide outward linearly in r, so the largest fitting r has a closed form — no search.
- Markings key off the CONFLICT extent, never off `reach`: a cross road of half-width h at angle θ crosses an arm's axis at `h/|sin θ|`, so a T-junction gets its bar at the stem and a skew arm contributes more than h. The keep-clear box is a rectangle from the same expression.
- **Winding is subtle here** (a recurring trap in this project): an arm's `(d, n)` frame is LEFT-handed (`n` is the left normal `(t.z, -t.x)`), the opposite of `hexa`'s right-handed basis, so `pushPost` walks its corner ring the reverse of `hexa`. Positions can be right and the mesh still inside-out; the test is **signed volume via the divergence theorem — positive is outward**.

**Water (`physics.js` `_stepWater` + `water.js` + `env.setWater`).** Opt-in via `world.water`; null for pre-water scenarios so the path is skipped entirely. The ground stays a single 220 m slab UNLESS a basin exists, in which case it becomes four land slabs around the hole plus a bed collider — and those land slabs are **20 m thick on purpose** (at 2 m a submerged car slid out from under them). Submerged bodies get saturating buoyancy (0.62 g, deliberately under neutral so wrecks settle instead of bobbing) plus linear and angular drag. Determinism rules match the driver controller: arithmetic only, thresholds compare **squared** magnitudes (never `Math.hypot`). Everything in `water.js` is render-side and cleared in `destroyRound`.
- **The visual ground never had the hole the collider did**, so the flat y=0 surface ran straight over the channel and the water was drawn entirely behind it. `punchBasin()` discards the rect **per fragment** on both the terrain and the ground disc (a geometric carve would land a ragged boundary on a polar mesh). A pin hashes bodies, not pixels, so this was invisible to the `water` pin.
- **The banks stay vertical** because the land slabs are vertical walls — a sloped visual bank promises a beach the sim will not honour. Riprap (armoured edge) is the correct answer that is also legal. Silt displaces **downward only** (the bed collider's top face is exactly `bed`).
- **Surface is `MeshStandardMaterial` + `onBeforeCompile`, not a raw `ShaderMaterial`** — it has to share the lighting/ACES/fog/shadow chain. The wave sum lands in `<beginnormal_vertex>` (NOT `<begin_vertex>`), because `vNormal` is built in `<normal_vertex>` which runs first. Wind drives amplitude and foam.
- **Crest foam thresholds are ABSOLUTE heights, never relative to `uAmp`** (P4): `smoothstep(uAmp·a, uAmp·b, vH)` makes `vH/uAmp` amplitude-independent, so "breaking" foam fires identically in dead calm and storm and every calm water body wears a lattice of blobs at the wave-interference maxima (caught by the scenes sheet on riverside/causeway/harbour — invisible to every pin). Absolute height + an amplitude gate: calm water has zero crest foam; whitecaps arrive with wind.
- **`world.water.v2`** is an opt-in on top of the opt-in: buoyancy and drag reach dynamic props and torn-off wheels (fixed order: props → each rec's `dyn` → debris). `buoy` is a per-CLASS choice about whether that class floats. Non-car entries fire **`onObjSplash`, never `onSplash`** — recorder.js resolves `onSplash` through `sim.cars.indexOf(car)`, so handing it a prop indexes `per[-1]` and throws inside the pre-sim.

**Scenery (`scenery.js`) — 184 kinds in 8 categories.** `{ id, label, icon, cat, build(r, M) → {g, bodies} }`, same contract as `vehicles.js`: same kind+seed ⇒ identical model AND identical collider recipe, zero `Math.random()`. Categories: Nature · Suburbia · Street & City · Signs & Traffic · Water & Coast · Mountain & Alpine · Rural & Farm · Industrial. **Adding entries is safe by construction** — the generators pick kinds from hardcoded dressing lists, so new models are inert until something names them, while the registry auto-propagates to showroom, counts, crosshair labels, smoke test and contact sheet. Heavy things use `fixedBody`, knockable things `dynGround` (which re-roots visuals so the body origin sits at H/2). Traps:
- **`cyl(mat, {axis: 'x'|'z'})` IS a rotation** — it sets `rotation.z = -π/2` (or `rotation.x = π/2`). Writing `.rotation.set(…)` or reassigning that component afterwards silently *undoes* it (a felled tree stands back up); only `.rotation.y` is safe to assign on an axis-built cylinder.
- **Nothing batches, so every mesh is a draw call.** Distance-read objects get an explicit LOD flag (`containerBody(…, near)`); the library's ceiling is ~68 meshes (`transmission_tower`).
- **You cannot subtract one solid from another here** — an open hull/bay/trough is built as a *shell* (floor + canted sides + ends).
- **Silhouette beats surface** — crags must break the outline, pushing past the top edge and off both ends.

**Props (`props.js`) & environments (`env.js`).** Props return `{group, bodies}` with *explicit* collider recipes, never parsed from geometry. Dynamic prop nodes re-parent to `sim.root` and sync from their bodies. Environments are strictly visual: preset palettes + deterministic decoration over the shared ground. **`ENVS` is 11** — `proving · salt · night · grid · city · dawn · dusk · alpine · coastal · desert · suburb` (G6 added `suburb`, the leafy residential noon — tjunction and the suburb worldgen preset had always named it and silently got `proving`, the exact ledger #5 class again). **An env id a topology names but `ENVS` lacks fails silently** (`apply()` falls back on an unknown id), so adding topologies means checking both ends; `city` exists precisely because two topologies always named it and silently got `proving`. The fix for such a gap is to ADD the missing preset, never to edit the director — that keeps every generated scenario byte-identical.
- **Adding a preset is safe by construction and is a 6-table edit** — nothing ever *picks* from `ENVS` (the director and worldgen name every id as a literal), so new entries cannot shift a generated scene and no pin moves. All six tables must be fed or something fails quietly: `PRESETS` (lights/ground) · `SKIES` (dome/sun/clouds) · `TERRAIN_FOR_ENV` (terrain.js) · `TERRAIN_VALUE` (env.js — baked colours no light can reach, so a low-sun preset must go well under 1) · `WATER_PRESETS` (water.js) · `WEIGHTS` (weather.js). `tests/weather.mjs` backstops the last one.
- **Presets are PLACES and TIMES, not conditions.** A storm is expressible on every preset and `WEIGHTS` decides where it fits; `dawn`/`dusk` carry the time-of-day axis (`el ≈ 0.12` gives raking shadows). Warm key against cool hemi is what sells low sun; warm *ambient* just reads as an orange filter.
- **Skybox**: a camera-following **shader dome** + sun/moon + deterministic stars, parallax cloud blobs and a scrolling cloud deck, all `fog:false, toneMapped:false`; `env.syncSky(camera.position, dt)` runs every frame. **Structure is per-PRESET, weather is uniforms**: `makeSky(id)` runs only from `rebuildSky()` (only `apply()` calls it); `syncSkyTint()` handles weather with zero-alloc uniform writes. The gradient is injected into `MeshBasicMaterial` (not a raw `ShaderMaterial`) so `toneMapped:false` + the material's colorspace chunk is the linear→sRGB path ACES gives everything else. The CPU still computes top/mid/hor with `mixHex` and passes them in, because `horizonNow()` also feeds the terrain's baked haze, `scene.background` and the fog tint — those four must agree. The shader adds only azimuthal sun glow and a horizon haze band. The cloud deck projects onto a flat layer (`d.xz / max(d.y, 0.12)`) so it foreshortens toward the horizon; higher cover *lowers* the fbm threshold so the deck closes up (fading in reads as dirty glass). Stars carry per-star size/brightness/colour off one seeded stream, brightness **cubed** so the distribution is faint-weighted.
- **The preset/weather split**: `apply(id)` rebuilds STRUCTURE and early-returns on an unchanged id; `applyWeather(w)` moves intensities/fog/haze/cloud and **never** early-returns. `apply` ends by calling `applyWeather` so a preset change re-lands the current weather. Fog near/far live in `env.state` and are scaled by `env.setFogScale(fk)` — don't set `scene.fog` directly.

**Terrain (`terrain.js`) — and it IS the ground.** The landscape around *and under* the play area, owned by env (`env.setTerrain(spec)` / `env.terrainField`), driven from `world.terrain` or a bare seed. It reaches r=0, so while it is up the old flat `CircleGeometry` disc **hides rather than being deleted**. Inside `playR` the height mask is exactly 0, so the play area is the same flat y=0 plane receiving shadows the same way. **Opt-in with null exactly like `world.water`.** One polar mesh (`rings × spokes`, radii packed as `u^1.7` so detail lands just outside `playR`). **Ring 0 collapses to the origin** (a triangle fan; emit its second triangle and `computeVertexNormals` returns a zero-length normal and a black speck). Displacement is masked by `smoothstep(playR, playR*1.35, r)`. Height is integer-hash value-noise fbm with an optional ridge fold and a domain warp (unwarped ridgelines read as corduroy). Vertex colours (rock/grass/scree/snow), non-indexed + `computeVertexNormals()` for the faceted read, one draw call. `makeHeightField(spec)` is exported so vegetation scatter and anything else that must sit ON the landscape queries the same surface.
**Drivable terrain** is opt-in on `world.terrain.drivable` (same null-opt-in). Until it, terrain was visual only: the collider is a flat 220 m slab whatever the landscape does. It samples the same field the mesh uses.
- **Rapier's heightfield layout is row-major `i*(n+1)+j`, i along x and j along z.** Measure with asymmetric ramps (the column-major reading is exactly the TRANSPOSE — plausible terrain that is wrong at every point).
- **Extent is sized off the ARENA, not off `rampTo`** (`rampTo` reaches only ~1.43 × playR; a relief topology keeps playR small). Resolution is a **cell size**, not a grid count, so widening the extent cannot quietly coarsen the surface under the cars.
- **The grid is memoised on the spec** (sampling is ~1.6 µs/call; a 194² grid cost 61 ms, more than a full reset). Cache is bit-identical to a cold build.
- Winding is the same trap as roads.js (spoke +angle, ring +radius → natural quad order yields a −y normal → backface-culled from above). Amplitudes are set against THIS world's scale (the far band gets a ×3.6 boost to break the skyline). A polar grid stripes unless each vertex is jittered within its own cell (ring 0 pinned).

**Weather (`weather.js`).** Per-scene weather. The RENDERING half is strictly render-side (reads nothing from the sim, writes nothing back). `rollWeather(seed, envId)` and `gripFor(wx)` are the other half and are **pure** (plain data, no THREE) — which is what lets director.js call them. `rollWeather` draws from its OWN rng stream (`'wx:'+seed`), so the same seed always shows the same sky and no existing stream shifts by a draw. 11 kinds, weighted **per environment** (a desert blizzard reads as a bug; `grid` stays clear so the diagnostic preset stays legible). `destroyRound` clears the sky and particles.
- **The SCENE carries its weather**: `generateScene` rolls it onto `world.weather`; main.js reads `sc.world.weather` rather than rolling its own. This is required once grip reaches the tyres — the recorder that priced the odds and the round the player watches must not read different weather. The roll survives in main.js only as the fallback for scenarios that never went through the director (crash mode, the pins).
- **The lighting is the feature; the particles are garnish.** `key/hemi/fill/exposure/envI` are MULTIPLIERS over whatever the preset authored. **`envI` (`scene.environmentIntensity`) is the biggest lever and easiest to miss** — the PMREM RoomEnvironment is a constant IBL flood no light intensity touches. **`hemi` goes DOWN in bad weather, not up** (albedo wins; on a white salt flat a high hemi produces a storm you can read a newspaper by). It rises only where the ground genuinely bounces (snow, mist); the key:hemi *ratio* still climbs.
- **Haze is a colour, not a grey.** Each kind carries `haze {hex, amt}`, and the sky gradient, `scene.background`, the fog tint and the terrain's baked horizon must all derive from the same `hazeTarget(preset, haze)`. Cloud and haze are separate knobs (cloud hides things *in* the sky; haze recolours the air).
- **Wet roads (`applyWetness`) do more work than the rain.** Roads only, materials from a per-build `matFactory` cache; terrain is left alone. Scalar-multiplying a linear-space `m.color` is an exposure change, not an HSL round-trip, so it is exempt from the sRGB rule.
- **Precipitation is two pooled systems with zero per-frame CPU** (fall/wind/wrap in the vertex shader against a 44×30×44 box that follows the camera; the box is small because density is count/volume). Rain is `LineSegments` (velocity-aligned streaks). **Flakes need ~2.5× the drop count** and size in metres through the projection (`aSize × uPix / -mv.z`) plus a near-fade. One draw call each.
- Weather is **frozen while the round is** (`update(dt, camera, paused)`) so hanging rain is the correct fiction and render-on-demand can sleep through the freeze.

**Weather grip.** Opt-in on `world.weather.grip`, the same null-opt-in: absent, `wxGrip` is exactly 1 and multiplying an IEEE-754 float by 1.0 is bit-exact. **It is TWO code paths**: `setWheelFrictionSlip` (scaled at build and again in the bent-wheel penalty, which re-derives slip from scratch) AND **braking authority scaled by grip** — without the second, `setWheelBrake` sits under the dry friction circle and peak decel is a flat ~13 m/s² from grip 1.0 down to 0.7, so wet roads had dry stopping distances. **The failure was directional**: friction slip alone flipped MORE markets at 0.82 than at 0.70 (chaotic, the "unfair randomness" the product forbids); scaling brake authority made it monotone (a player can learn it). **A hash moving is not evidence** — `settleMarket` over the real market set is what decides the design. Grip is per-KIND (snow barely wets asphalt but is the slipperiest; dust is dry but grit costs grip), clamped 0.35–1 (below ~0.35 cars understeer off the road before the incident), guarded with **`Number.isFinite`** (NaN passes a `typeof` test, `clamp` propagates it, and it reaches wasm as a bare panic). **CALIB has no weather axis** (measured: wet-vs-dry gap is inside estimator noise, so splitting the bucket would fit noise not signal).

**Vegetation (`vegetation.js`).** Instanced scatter over the terrain ring, render-side: reads `env.terrainField`, writes nothing. No colliders, masked to `r > playR * 1.06`. `veg.build(field, seed, {density, value})` from `startScene`, `veg.clear()` from `destroyRound`.
- **One InstancedMesh per KIND, not per (kind, material)** — each prototype is merged into ONE buffer with the per-mesh material colour baked into a **vertex-colour attribute** under a single white `vertexColors` material; per-plant variety comes from `instanceColor`.
- **Scatter is near-biased (`u^1.9`), not area-uniform** (area grows with r², so uniform put ~80 % of plants in the outer third and the roadside was bare). Species carry a `reach`.
- **Sway is `h² · amp` in the vertex shader** (injected after `#include <begin_vertex>`; the quadratic IS the trunk mask). **Verify sway with a pixel diff across ~0.75 s, not by reading uniforms** (the first amplitude was 4× too small and indistinguishable from a still frame).
- Plants carry baked colours no light can reach, so `build` takes `env.terrainValue`. Frozen during the freeze and under reduced motion.

**World generators (`worldgen.js`).** `generateWorld(preset, seed, {maxProps, maxRoads})` (suburb / city / highway) emits plain scenario content — G6 also `junctions` (apron seams). One rng stream per `'w:'+preset+':'+seed`; placement walks the spline with lateral zones chosen so nothing overlaps by construction. Props push identity-first (anchors → lots → filler) and trim to `maxProps`, so smaller budgets drop filler, never identity — but the same seed gives a *smaller subset* on mobile, and that trim order is part of each generator's design. G6 scale-up: suburb half 52 → 78 with two cul-de-sac side streets (**short stubs < 55 m so `lanesOfRoad` never lifts a lane off them** — pure visual streets with their own houses, seamed by apron junctions), city block ~1.45× with a second rank of towers, arenas 190/165/300, director budget `maxProps: 84`. The prop-clamp half-extent is per-generator (`EXT`), not the old hardcoded ±99. Gotcha: the city loop's left normal points OUT of the block, so outside-ring props use positive lateral offsets and the plaza uses centroid math.

**Effects (`fx.js`).** Strictly render-side; reads sim state and hooks, never writes back (fx has its own `makeRng('fx')`). Pooled zero-alloc systems: four shader point clouds (sparks / normal-blend puffs / fire / water spray), instanced glass shards and paint chips, a ground-decal ring buffer, an impact-flash PointLight, and camera shake applied around `renderer.render` only then undone. Struck props start *standing* emitters keyed on `props[i].spec.kind`, anchored to the prop's SPEC position (the body tumbles away while the sheared main stays planted). Fire spread is render-only ignition in an fx-local map (`frontDmg` is sim state; fx must never write it). `initFX(scene, {small})` halves pools on small screens; `attach(sim)` OVERWRITES the six hooks and re-keys `perCar`; `update(dt, camera)` returns busy so render-on-demand still sleeps when idle.
**Audio (`Sfx`)** is procedural and zero-asset. The engine bed is a second sustained voice alongside the scrape loop, working because it occupies a different band (two detuned saws at 55–190 Hz under a 340 Hz lowpass vs the scrape's 1050 Hz bandpass). The **rain bed (P4)** is the third: broadband hiss high-passed at 2.8 kHz under a 9.5 kHz lid — above both, fighting neither — fed by `setRain(intensity)` from main.js (rain kinds only; snow falls silent), zero while frozen or reduced-motion, and `_duckRain` dips it under crash one-shots. `setRain` remembers a level set before the gesture unlock and `unlock()` re-lands it. UI sounds go through `_ping`, which does NOT consume a crash one-shot slot. The context needs a user gesture, so `?crash=N` boots register a one-time pointerdown unlock.

**Incident engine (`director.js`).** `generateScene(seed, d)` deals a complete deterministic round from separately derived rng streams per facet (`'scn:'+seed+':topo'/'incident'/'cast'/'dress'`, G6 adds `':env'`). **22 topologies** (bespoke signalized intersection + suburb/city/highway from `generateWorld` + causeway · switchback · schoolzone · tramcrossing · parkinglot · roundabout · boulevard · tunnelmouth · industrialyard · tjunction · overpass · cloverleaf · forestroad · mountainpass · canyon · coastalcliff · riverside · harbourramp); **~118 incident templates** (G6: 28 → ~118 via motif FAMILIES — bespoke motifs like uturn/overtake/race/stuckthrottle/yieldfail/islandhop/quayplunge/storefront plus `swerveAt`/`queueSlam` factories and `reskin(base, o)` variants that re-run a proven base and retouch the result). Each template may carry `cal`: **`meta.calKey` groups variants under their choreography family** so CALIB cells stay thick — markets.js and tools/calibrate.mjs price/aggregate on calKey, `meta.template` stays the variant identity. A module-load assertion rejects any template naming an unknown topology. **Every scene now carries**: an env drawn from a per-topology `ENV_POOLS` list (never proving/grid — those are dev surfaces; P4 hoisted the table to module scope with a load-time assert against `WEATHER_ENVS`, so an unknown env id throws at import instead of silently rendering on the proving ground — ledger #31), a visual `world.terrain` ring (preset from `TERRAIN_FOR_ENV`, amplified via `ampK`; relief topos keep their own drivable spec; P4 adds `valueK` — a render-side brightness hint env.js folds into its baked-value expression clamped ≤ 1, set only for mesa+dawn/dusk where low-sun value × weather cloud rendered the bowl floor void-black), and `world.aware: true`. Forced casts: `_type` (exact REG id: tram/moto/tractor…) wins over the pool draw (the draw still happens so streams never shift); `_pool` gained EMERG/BIKE/SLOW; heavies/`_type` are only forced on straight-lane topologies (the bendy guard's argument). mountainpass/canyon/forestroad set `world.terrain.drivable` + gravel-verge / guardrail style bits; mountain/canyon/forest/coastal/riverside set `wxGrip`. Topologies that meet at grade emit `junctions`. Adding one entry to the topology or template array changes which scene every existing seed deals (both are `rng.pick` over the array), so content additions are inherently pin-moving — that is why pins move on purpose and CALIB is regenerated when they do. **Everything goes wrong at exactly tick 600.** Hard-won invariants:
- **Nothing may collide before tick 600.** A conflict scrub staggers crossing arrivals, then a spawn-overlap net runs BOTH before and after the scrub (the scrub moves cars).
- Head-on templates (`needsOpp`) only deal where opposite-lane geometry exists, and the victim's arrival window accounts for COMBINED closing speed.
- Heavies are barred from fast bendy plans (`v²·κ > BENDY_MAX`, empirically 2.0) — a long semi off-tracks across the centreline on a sustained gentle bend, and the `bendy` guard measures the tip, not the length-driven crab. Curvy/relief topos also bar heavies via `NO_HEAVY`.
- **`sortLanes` quantises its length key to cm and tie-breaks on build order** — the two directions of a symmetric road are equal to within float noise, and a raw `b.len - a.len` let `sqrt`'s last bits decide, so node and the browser sorted a causeway's lanes oppositely and generated **mirrored scenes from one seed**.
- **`place()` may drop below `MIN_V` to hit its arrival tick exactly** (templates aiming past T=0 need more road; the old floor clamped `s0` to the lane start so the car arrived EARLY and rear-ended a still-parked victim).
- The near-miss shift is capped by the room the victim can give up and still reach its anchor after T=0. The spawn-overlap net cannot catch this class (the cars never overlap at spawn, they converge early).
- A signalized T-junction is impossible with this lane model, so `tjunction` is a give-way SPUR with no through traffic (the incident on the main road). The roundabout emits four 2-arm **aprons**, one per stub mouth (where the real seam was).
- **The roundabout is a real roundabout (G6)**: lanes are enter → circulate-half-the-ring → exit JOURNEYS at ring radius 23 (inside the driver's certified 19–48 m band; v²·κ ≈ 3.4 bars heavies via the bendy guard). Each journey carries a unique `road` tag, so a one-way ring has no oppOf partner and head-on templates skip it by geometry. The topology publishes `yieldZones` (mouth conflict circles + give-way arcs); every AMBIENT car on a matching lane gets `drive.yields` — **never a lane an essential car occupies** (same exclusion as signal stop lines, same plow-into-the-hold failure otherwise). **`crossOf` is unusable on MERGING paths** (its min-distance point lands anywhere along the shared arc — timing a pair to it converges them nose-to-tail pre-600): the yieldfail template aims at the MOUTH via the yield zone instead.
- **Scenes must CONCLUDE (G6)**: background traffic paths are truncated past their anchor (`trimDrive` — ambient +65–110 m, queues +55–85, decoys +85–125), and every lost-control choreography hands the wheel back (`off: false` + a stop) — a car left in `off` orbits on the 0.1 drag brake for thousands of ticks. This took ran-to-cap from 21 % back under the sweep's 10 % budget.
- **`scrubConflicts` rebuilds a pushed-back car's `drive.pts` from the FULL lane** — everything measured along the old path start must move with it: stop/yield arcs shift by `back`, and the trim is re-applied (`_anchor + 90`). A stop kept at its old arc read as "passed" 26 m early, latched done, and the car sailed its red into cross traffic.
- Dressing is scrubbed off lane corridors, out of big-prop footprints (a tree inside a house collider gets ejected and falls across the road), and away from car spawns.

**Traffic signals (`signals.js`).** A program is plain data and `signalAt(prog, arm, tick)` is a **pure function of the tick** — no rng at eval time, no state between ticks, integer arithmetic. It settles money (drivers obey it) and a backward freeze-scrub must get the lights the forward run had. Arms are keyed by the string lanes carry as `road` (`'ew'`/`'ns'`); programs live on `junctions[i].signal`; `physics` evaluates, `main.js` mirrors the aspect onto lamps (tagged `userData.sigLamp`, `noMerge`).
- **An unknown arm returns GREEN** — decide "is this arm governed?" BEFORE the stage walk (the walk returns early, so an arm appearing only in a later stage was wrongly reported red).
- **Intergreen must cover junction clearance** (amber + all-red must exceed the ~1.6 s a car takes to cross, or cars legitimately mid-junction are T-boned). G6: all-red 80 → 110 AND the driver's amber rule hardened (go only when a HARD 4.6 m/s² stop cannot make the bar; stopping cars use a 4.4 ramp) — the big casts exposed the flowing-arrival case, where a far-edge amber-committer was still clearing as a cross car arrived at speed on its fresh green.
- **Phase by SEARCHING the offset, not by pinning one (arm, tick)** — an intersection incident involves actors from both arms, so a two-stage cycle just needs the right offset.
- **Obedience is all-or-nothing per scene** — essential cars never carry stop lines (their run-up is budgeted at free-flow cruise), so the signal is arranged *around* them; if no offset covers every actor, ambient traffic ignores the signal (half-obeying breaks the tick-600 rule). Stop lines stop binding at `INCIDENT_TICK + 150` (else obedient traffic is perpetually restarted by a fresh green and the scene never settles). The driver holds at the bar via `v² = 2as` plus a firm hold; amber is a dilemma zone, not a second red; the done-latch lives on the DRIVER, never on `drive.stops` (the scenario is shared between recorder and live sims).

**Driver controller (in `physics.js`).** Cars with `spec.drive` get closed-loop pure pursuit: arithmetic-only (`+,-,*,/,sqrt,min,max,abs` — no transcendentals, so bit-deterministic across engines), anticipatory `v² = 2as` braking, `end: 'stop'|'coast'`, sparse cmd timeline, post-crash shock (damage > 6 → brake and hold), and **ACC car-following for ambient cars** (path-probe detection, a same-direction gate `dot > 0.2`, capping to the leader's TRUE velocity projected on own heading, full attention only before the incident tick). **G6 awareness (`world.aware`, null-opt-in like grip/water)**: past the attention bound an ambient driver is not blind — after a personal reaction delay (`drive.aw`, rolled per car by the director) full-range detection RETURNS with a following margin, so the ambient cast brakes for the wreck instead of filing into it; panic range still applies during the reaction window, and essential cars (no `acc`) keep their scripted late brakes. **G6 give-way lines (`drive.yields`)**: hold at an arc like a red light while any MOVING car (speed² > 1.44) occupies the conflict circle — a held yielder is itself under the threshold, so two waiting mouths can never deadlock; `until` releases post-incident. Driven cars get real brake authority (`brakeK` 0.055 vs the legacy 0.02, which keeps old hashes frozen), **scaled by `this.wxGrip`** (the tyre model alone does not bind in the braking range). `sim.stopAt` freezes on an exact tick; `colToObj` maps collider handles → `{kind}` for impact attribution.

**Pre-sim recorder (`recorder.js`).** `recordScene(...)` runs the scenario headlessly to rest and returns `{events, summary, restTick, ticks, hash, tracks}` — the log the whole betting game settles against. **Rest is measured by DISPLACEMENT, never instantaneous velocity** (three steady states read as motion forever — raycast suspension holds a constant +lv.y, buoyancy holds another, a stuck car under driver control yaws in place). The threshold is derived, not tuned: 0.6 m of drift across a 75-tick window caps a body under 0.5 m/s, so no two bodies below it can close hard enough to log a contact (`TOUCH_DV` is 1.4). Deliberately exempt from the gate: disturbed **props** (markets latch on first move, a knocked cone rolls forever) and **sunk** cars. Events are fixed-order read-only scans inside the step loop; a new event kind MUST get an `EV_CODE` or it hashes as `undefined`. The FNV event hash is the determinism fingerprint (bit-identical node↔browser). `tracks` are coarse per-car positions every 10 ticks (not hashed) — used for scrub UIs and to place the camera rig on the incident.

**Betting core (`markets.js` + `economy.js`) — headless-first.** `generateMarkets(scene, {labelOf})` is **outcome-blind by construction** — it never imports recorder.js; inputs are the scenario plus the static `CALIB` table of empirical hit rates keyed kind → template-FAMILY → bucket (G6: the lookup key is `meta.calKey || meta.template`, so ~118 template variants price through ~47 family rows and cells stay thick; `specialFor` switches on the family too, and `carSunk` joined the settle kinds for the harbour plunge). Every market carries its `cal` bucket tag so the harness measures exactly what the generator prices, plus `p` (the calibrated probability) and `oddsH` (decimal ×100 integer). `oddsFromP = (1 − marginFor(d)) / p` clamped ×1.10–×50; margin eases 6 % → 2 % at d ≥ 8. **Offering rules**: markets with `p > OFFER_MAX` (0.80) clip at the odds floor and pay the BETTOR, markets with `p < OFFER_MIN` (0.05) are below the pricing clamp and unreadable — neither is offered; the headline pair is exempt (the spec requires both sides) and gets a ×1.03 floor. `settleMarket(market, rec)` is the only outcome reader. Two calibration rules that cost real debugging: `tools/calibrate.mjs` MUST pass `{ all: true }` (offering is a function of CALIB, so calibrating over only the offered set does not converge); and **there is deliberately no difficulty axis on CALIB** — two marginal corrections both made the gate worse, and the real cause of a bad gate was a stale table, so **regenerate CALIB first** and reach for a difficulty correction only with a calibration sample bigger than the Monte Carlo validating it.
`economy.js`: injectable store (localStorage in app, memory in tests), profile v1, slip = singles + one parlay (integer stakes ≥ $1, total ≤ bankroll — the bankroll cannot go negative by construction), `placeSlip` deducts at bet time, `settleRound` pays ⌊stake·oddsH/100⌋ (stake included), pushes the ledger, advances the campaign, and handles ROCK BOTTOM. **Achievements** (`achievements.js`) evaluate only on money rounds (Exhibition seeds are re-runnable, so unlocking there would farm badges) and unlocks are append-only and idempotent. The **daily seed** is a normal money round with a date-derived seed (no profile key, so everyone shares the scene), where the settled-seed ledger enforces one attempt; streak arithmetic parses the key at **midday** so a DST shift cannot move "yesterday".

**Game UI (`betui.js` + `povcam.js`).** `betui.js` owns every pixel of the betting layer and NO sim state — main.js drives the lifecycle and calls `openRound / setPhase / tickLive / settle / closeRound`; the only callback out is `onNext` (deal the next round). Resuming time at the freeze is main.js's `#fzGo` button, not a betui callback. **Live chips are a preview, not settlement**: `buildLiveIndex` precomputes per market the tick it becomes definitively won/lost, so a chip flips the frame the player sees it; by construction the state at `restTick` equals `settleMarket`, and `settle()` overwrites every chip from the authoritative report. Market rows show odds + implied probability + to-win (not a bare ×2.50); groups are collapsible and the aggressor's group is pinned as "the one to watch". During resolve the slip diff-updates leg classes rather than rebuilding `#sliplegs` innerHTML every frame; the localStorage draft write is debounced. The scene **summary is dismissable** (a floating chip reopens it) so the player can explore the settled scene with every camera angle unlocked. Bugs worth not reintroducing: re-rendering `#mklist` innerHTML on a tap resets scrollTop (tap path uses `syncMarketRows`); **there is deliberately no tap-anywhere-to-resume** — `#fzGo` and the space key are the only exits, and the freeze layer is a pass-through box; a half-built parlay sits at one leg and blocks PLACE rather than silently dragging in another pick.
`povcam.js` builds the deterministic camera rig (`'pov:'+seed`): dashcams on flagged cars (the aggressor rarely gets one — riding it hands over the read), CCTV pole, chopper orbit, witness tripod; `povBudget(d)` prunes coverage, but the kept set INTERLEAVES fixed and dash cameras so a wide shot always survives at every difficulty. The rig is built around the INCIDENT location (from the tape), not the tick-0 spawn centroid. Everything unlocks at `phase === 'done'`. **Cameras**: Orbit (default) and freecam (WASD/pointer-lock on desktop, joystick + look-drag on touch), toggled by the cam button / `C`; scripted POVs own the camera outright and stand orbit/freecam down. `setCamMode` clears `activePov` on entering freecam (or a scripted POV keeps overriding the camera every frame). `V`/`Shift+V` cycle the rig. Camera MOTION lives in povcam.js, the LENS treatment (grain, vignettes, letterbox) is CSS on `#povfx`. The fit-tween and the resolve auto-follow both move `controls.target`, so the auto-follow stands down while a tween is in flight (`camT < 1`) or they fight.

**Freeze scrub — the filmstrip (`buildStrip`/`poseStrip`/`seekPreview` in main.js).** There is no rewind in a rigid-body world, so a backward seek used to `sim.reset()` and replay from t0 — ~30 ms of rebuild plus up to 600 steps *per drag frame*. Entering the freeze now captures the last 10 s of **visual** state once (`buildStrip`) and a seek becomes a pure pose: **0.016 ms**, no physics, no rebuild, no allocation. Captured per tick is exactly what `syncVisuals` interpolates (chassis pose, steer, suspension, wheel spin, dynamic props); debris is not captured (nothing may collide before the incident tick). The payoff is structural: **the sim is never rewound at all** — it stays parked on the incident tick, so `resumeRound` cannot hand the player a scene the odds were never priced on (`hashState()` is bit-identical across every seek). Capture order matters — `buildStrip` calls `reset()`, so fx must `detachSim` first, `targetMap` must be rebuilt from the fresh meshes, and the six hooks are nulled for the capture pass then restored by `attach` + `hookRoundCinematics`. **Every difficulty builds the strip** (the freeze is offered at every level — the difficulty is the READ, not a denied study beat).

**Quality tiers.** `applyQuality(q, save)` owns DPR, `shadowMap.enabled`, shadow map size, vegetation density and the precipitation draw-range budget, and it **persists** in `profile.settings.quality`. The material recompile happens only when `shadowMap.enabled` actually flips (the one change that alters the shader). The boot call passes `save: false` and is guarded by `qualityLive`, because it runs at module scope *above* the `let round` / `let profile` declarations (reading either there is a temporal-dead-zone throw).

**Loading + motion.** `makeProgress` is ONE component reused three ways (boot preload · per-round pre-sim beat · scene loading) against shared `.pbar`/`.pfill` markup. **Reduced motion is a live setting, not a boot constant**: the OS media query seeds it and the in-game toggle can turn it ON, but the toggle never overrides an explicit system preference back to "full". It mirrors onto `body.reduced-motion` so CSS, the camera tween and the bankroll count-up honour one switch. P4 added the render loop's share: precipitation parks exactly like the freeze does (hanging rain is already the shipped fiction, and a paused volume lets render-on-demand sleep) and camera shake — the one true vestibular trigger — is skipped entirely (both `applyShake` and `undoShake`; symmetric, since `shakeOff` is zeroed by the last undo).

**`main.js`** owns the shell: renderer (ACES, PMREM RoomEnvironment, shadow frustum refit per scene size), camera fit/tween, showroom builder, menu/pause/fullscreen/orientation, settings, boot, and the round lifecycle. Critical: **never run `controls.update()` while freecam or a scripted POV owns the camera** — OrbitControls re-aims at its target even when `enabled=false`. Crosshair targeting uses a per-round `Object3D → group id` map, raycast from screen centre in freecam or the pointer in orbit.
**Fog scale follows the LIVE camera during a round** (`syncFogScale`), not the distance `fitCamera` picked: the metric is `fogBox.distanceToPoint(camera.position) / 8` clamped `fogFloor()`–20, against the round's bounding box cached at fit time. **The floor is not 1** — it is `groundRadius / 28` clamped 1.5–6, because the preset fog distances were authored when the world was a 90 m disc. Distance to the box, **not to its centre** — a dashcam on a car at the far end of a 150 m highway is 120 m from the centroid while sitting 2 m from what it films. `distanceToPoint` is 0 anywhere inside the scene, so any camera among the cars gets weather at full strength — which is also what keeps weather **fair** (backing off to read the whole scene is the same motion that pushes the fog away).

**Conventions everything relies on:**
- Vehicle space: forward = **+X**, ground = **y 0**, width along z. Builders return a `THREE.Group`; `buildVehicle()` recenters on x/z.
- Determinism: same seed+type+paint ⇒ identical vehicle. Type choice, build randomness and naming each use a *separately derived* rng (`makeRng('t:'+seed)`, `'b:'+seed+':'+typeId`, `'n:'+…`) so forcing a type doesn't shift the build stream. Never call `Math.random()` inside builders — always the passed `r`.
- **Merging (`lib.js` `bakeMerged` / `mergeByMaterial`)** flattens a STATIC subtree to one mesh per material. Sim-neutral by construction: colliders are explicit recipes never parsed from geometry, and fx reads `rec.spec`, not meshes. `byParams: true` keys on material *parameters* (matFactory dedupes within one build, so identity-keyed merging can never join two builds' "black rubber"). **Never merge a vehicle in the sim** — deform.js displaces per weld group with per-zone stiffness. `userData.noMerge` opts a mesh out. **Two colour paths**: `opt.color` BAKES each mesh's material colour into new vertex colours; carrying an *existing* per-vertex colour attribute is automatic (mergeByMaterial never asked for colours, so geometry that had them lost them silently and rendered black against a `vertexColors:true` material). Precedence is mesh colours → material colour → white.
- **The geometry kit (`lib.js`)** produces the look; prefer it over raw boxes. `slab(mat, {…})` is a chamfered-box frustum carrying `userData.pt`; `faceQuad(pt, side)` + `subQuad` + `quadPrism`/`panesOnQuad` place thin plates flush on any face including sloped ones. `faceQuad` assumes slab-shaped `pt` — on `wedge()` (plan-tapered) faces the panes skew, so use round `cyl` lamps there. `hexa(bottom4, top4)` is the underlying 8-corner solid; winding order is exact — copy an existing call. All geometry is non-indexed + `computeVertexNormals()`; materials come from the per-build `matFactory()` which dedupes by parameter key.
- **Family builders (`families.js`)** — `car()`, `truckFront()`+`chassis()`, `van()`, `bus()` — are config-driven; most of the 92 registry entries are parameter sets. Only genuinely novel shapes (F1, dragster, excavator, tram, bikes) get bespoke builders.
- **Registry (`vehicles.js` `REG`)**: `{ id, label, cat, build(r, M, ctx) }`. `ctx.paint` must win over any `paintHex` default. Adding a REG entry updates the UI, counts and smoke test automatically; optionally add a flavor name in `names.js`.

## Product rules (do not quietly change these)

- **Money**: integer dollars only; min bet $1; max = bankroll; payout = ⌊stake × odds⌋. Singles ×1.10–×50, parlay capped ×500. House margin ~6 % easing to ~2 % at d ≥ 8 — hard reads pay nearly fair. **Never move `marginFor` to make a gate pass**; that is changing the product to fit the test.
- **Anti-cheese**: campaign seeds hidden until settlement; each seed settles winnings **once per profile** (ledger); custom, shared and replayed seeds always run as **Exhibition** (playable, zero bankroll/ledger/stats/achievement mutation).
- **Difficulty** controls incident subtlety, cast size and similarity, camera coverage, near-miss (≥5) / decoy (≥6) / multi-incident (≥7) rates, and odds spread. **The freeze is offered at every level** — the difficulty is the READ, not a denied study beat. Never unfair randomness: every outcome must be readable in the preview if you look at the right thing — d controls how hard "the right thing" is to spot. **Bet quality is "not obvious, not impossible"** — `OFFER_MAX`/`OFFER_MIN` keep the offered band decidable. Campaign draws d on a geometric rarity curve (`P(d) ∝ 0.62^(d−1)`).
- **Persistence**: `localStorage["crashbet.profile.v1"]`, versioned JSON — bankroll, campaign cursor, current round + slip draft, settled-seed ledger (ring buffer, 500), stats, achievements, daily streak, settings. Boot always resumes the last unfinished round. Reset wipes after a confirm.
- Vehicles and property only. No people, no gore, ever.

## Gotchas learned the hard way

- **`box()` and `cyl()` return INDEXED geometry.** "All geometry is non-indexed" is true of the hexa/slab kit and false of three's primitives — only `IcosahedronGeometry` (`sphere()`) is non-indexed. Anything that reads a vertex buffer directly **must walk `geometry.index`** (copying `position` in buffer order builds spikes across the scene, quietly wrong on small distant objects). `bakeMerged` is the one place this is handled — go through it.
- **A hidden subtree still costs a full matrix walk.** `visible = false` is a render-time cull; `scene.updateMatrixWorld()` recurses into hidden children regardless (the showroom's 7 321 nodes cost 2.3 ms until it was *removed from the scene*). `matrixWorldAutoUpdate = false` does NOT stop that recursion; the knob that saves work is `matrixAutoUpdate = false` (`freezeMatrices`).
- **Measure, then optimise, then measure again — this machine's frame timing has a ±1 ms noise floor.** Only an interleaved, warmed-up, median-of-N comparison separates a real win from the noise (a shadow-caster-culling optimisation was measured this way and **rejected** — disabling the shadow pass entirely changed nothing outside noise).
- **Color math must stay in sRGB**: pass `THREE.SRGBColorSpace` to `getHSL`/`setHSL` (linear-space HSL turns saturated reds salmon).
- Look-tuning baseline (don't casually change): exposure 1.18, key 1.7, hemi 0.55, fill 0.45; glass `#1b2836` rough 0.32 env 0.85. Brighter key lights wash colors through ACES. Weather multiplies these; it never replaces them.
- **You cannot darken a scene with the three lights alone.** `scene.environment` (PMREM RoomEnvironment) is a constant image-based flood that ignores every light intensity — the knob is `scene.environmentIntensity`. Symptom: killing the key yields a bright scene with black shadows.
- `Mesh.position` is a read-only accessor — never `Object.assign(mesh, { position: … })`.
- Tall truck beds must pass `stacks: false` to `rigidTruck` or the exhaust stacks clip through the bed.
- Old vehicles must be disposed via `disposeGroup()` before replacement; Rapier objects need explicit cleanup (`CrashSim.dispose()` frees world, event queue and controllers — never drop a sim without it, wasm leaks silently).
- Physics wheel discovery keys off `userData.wheel` set in `parts.js` `wheel()` — a bespoke builder rolling its own wheel cylinders gets no suspension. Prefer `P.wheel`/`P.axle`.
- Prop bodies whose `node` IS the prop group must treat the node-local pose as identity in `_addPropRig` — reading `node.position` after the group was placed double-applies the offset.
- **A calibration correction must be measured as observed/expected against the model's own predictions**, never as a marginal rate (a marginal rate re-counts whatever the model's other keys already explain). If the 95 % interval on your calibration sample is wider than the effect, the sample is too small to act on.
- **Never compare crash-mode replays after N wall-clock frames — compare at the same TICK** (auto slow-mo is on a 5-second real-time cooldown, so the same frame count executes different tick counts).
- **Never animate `opacity` from 0 with `animation-fill-mode: both`, or `max-height` from 0, on anything load-bearing.** An animation that never runs (reduced motion) holds its 0 % keyframe forever and the content never appears. Drive visibility from a class + JS timing, or animate `transform`.
- **`.mbtn` sets `width: 100%`, so it cannot go into a flex row unchanged** — add `width: auto` when flexing it.
- **PWA version skew**: a service worker that serves a stale `index.html` next to new JS throws on the first missing new element and bricks boot. Bump `sw.js`'s `CACHE` on every asset change and keep `ASSETS` complete; guard new-element access at mount time. During dev, unregister the SW + clear caches if an edit does not appear.
- **The embedded Browser pane** reports `document.hidden`, never fires `requestAnimationFrame`, freezes CSS transitions, and its `ResizeObserver` on `#stage` never fires (so `#stage` can stay 0×0 while everything else works — re-run `renderer.setSize` + `camera.aspect` + `updateProjectionMatrix()` before a screenshot). Drive the game with `__app.pump(now)`; a "wrong" `getBoundingClientRect` may be a transform frozen mid-flight; `computer{action:screenshot}` may time out (rasterize via `foreignObject`). Do NOT conclude the sim/camera/HUD is broken because nothing moves on its own.
- **CSS and JS edits need a hard reload in the preview pane** — the stylesheet and module registry are cached, so `await import('/js/x.js')` returns the instance already in the page, not what is on disk. Hard-reload before any in-page probe of edited code.
- `?crash=N` boots with no user gesture, so WebAudio stays suspended until the first pointerdown — not a bug.
- `run.bat`'s port cleanup uses `findstr /c:` deliberately (a bare space in a `findstr` pattern means OR).
- Never round-trip files through PowerShell `Get-Content`/`Set-Content` (UTF-8 mojibake) — use the Write/Edit tools.
- **Never push unless explicitly told.**
