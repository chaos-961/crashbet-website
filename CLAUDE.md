# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

"Crash Bet" — a deterministic crash-physics sandbox on top of a seeded procedural low-poly 3D vehicle generator (92 archetypes), on its way to becoming a **physics-betting game** (see "Crash Bet — the game (G1–G5)" below; that section is the product spec and the roadmap). Vanilla JS + Three.js r169 + Rapier 3D 0.19 (`libs/rapier3d-compat.module.js`, WASM inlined). No build step, no package.json, no framework. Three.js is vendored in `libs/` (import map in `index.html` maps `three` → `./libs/three.module.js`), so the app is fully offline.

**Current app shell (game era)**: boots to a main menu whose buttons are **gated behind a boot preloader** (physics module + Rapier wasm + fx pools + showroom build warm up behind a progress bar; `preloaded` flips true at 100 %). "Start Game" goes fullscreen (landscape-locked on mobile, best effort) and opens the **showroom** — every model (92 vehicles · 102 scenery · 4 props · 2 road pieces) flow-laid on the ground disc, orbit/pan/zoom to browse. **🎬 Play a Scene** (or `?scene=<seed>~<d>`, both parts optional, dev/no-fullscreen) deals a G1 director round: themed loading beat over the headless pre-sim → 10 s live preview with a countdown ring → hard freeze on the exact incident tick (tap/space/▶ resumes; d ≥ 8 skips the freeze) → resolve to the recorded rest tick with the crash-test cinematics → outcome card. **💥 Crash Test** on the menu (or `?crash=N`, 1-based scene index, dev/no-fullscreen) enters crash-test mode: 8 seeded crash scenes running the full damage + FX stack, with Replay / Next / Slow-mo / Mute buttons (R / N / T) and a damage-ticker HUD. A top-left hamburger opens the pause menu (Resume / Settings: environment + quality / Fullscreen toggle / Leave). `?play=1` skips the menu without fullscreen (dev/headless). **The unified editor is mothballed, not deleted**: `js/editor.js` (spawn/gizmo/inspector/URL codec/worldgen UI) is intact but no longer imported by `main.js`, and its DOM was removed from `index.html` — re-wiring it needs its DOM back plus the `initEditor(ctx)` call. Editor-era URL params (`scene`/`car`/`prop`/`road`) are currently not parsed by the shell. The editor-era CSS is kept at the top of `css/style.css`; the game shell's styles are the marked section at the bottom.

## Running & testing

- Serve statically over HTTP (ES modules won't load from `file://`): `run.bat`, or `npx http-server -p 5183 -c-1 .` — then open http://localhost:5183. A `.claude/launch.json` config named `garage` exists for the browser preview.
- **Smoke test**: open `?smoke=1` — builds every registry type × 4 seeds at boot and logs `SMOKE DONE: N/N ok` (plus per-type stack traces) to the console. Run this after any change to `js/vehicles.js`, `js/families.js`, `js/parts.js`, or `js/lib.js`.
- **Determinism self-test**: open `?simtest=1` — runs seven scenarios (`TEST_SCENARIOS` in physics.js: the legacy 3-car crash; an extended one with gravity 20, arena walls, ramp/boxes/pole props and per-car mass/grip/rest/crumple/delay/rolling/brake; a scenery one with seeded fixed+dynamic scenery props; a roads one with two spline roads whose curbs get clipped at speed; a **carnage** one tuned to tear wheels off and shatter glass — it pins the damage-model era, since debris bodies enter the world mid-run; a **director** one that replays a full generated round (`generateScene('pin-1', 4)`) — it pins the driver controller AND the scene generator; and a worldgen one that runs a full generated suburb — which also pins the generator, since layout drift changes the hash) twice each (300 fixed steps), FNV-hashes every body transform per step (debris included) plus the final crumpled geometry, logs `SIM DETERMINISTIC: ok/FAIL`. Run after any change to `js/physics.js`, `js/deform.js`, `js/props.js`, `js/scenery.js`, `js/roads.js`, `js/worldgen.js`, or `js/director.js`. The hashes are stable across node and browser (same wasm f32 math). - **Headless physics testing in node** (much faster than the browser loop): `node_modules/three/` is a gitignored shim so the whole vehicle+physics chain imports in node. Import modules via `pathToFileURL`, build a `CrashSim`, call `stepOnce()` in a loop, assert on `body.translation()` / `hashState()`. See the CrashSim API in `js/physics.js`.

The pattern that works even when the preview tab reports `document.hidden` (rAF suspended, screenshots time out, canvas rect 0×0 — common in the embedded Browser pane on this machine): `await __app.startGame(false)` (or `startCrash(false)`), then `renderer.setSize(w, h, false)`, position the camera, synchronously `renderer.render(...)` + `canvas.toDataURL()`, and POST the data URL to a local sink server that writes a PNG you can view. **`__app.pump(now)` runs one real frame of the app loop** (camera tweens, crash sim + fx, auto-follow, HUD, render) with a synthetic clock — loop it with `now += 1000/60` to drive the whole game headlessly instead of re-implementing the loop. DOM overlays (menu/pause) can be rasterized headlessly via SVG `foreignObject` with the stylesheet inlined.

## Architecture

Module chain (each imports the previous): `js/lib.js` → `js/parts.js` → `js/families.js` → `js/vehicles.js` → `js/main.js` (+ `js/names.js`). Sim side: `js/deform.js` + `js/props.js` → `js/physics.js` → `js/main.js` (+ `js/env.js` + `js/fx.js`, the render-side effects layer). Game side (G1): `js/director.js` (imports roads + worldgen, NEVER physics) → `js/recorder.js` (imports physics + director + roads) → `js/main.js`; physics.js imports director.js only for its pinned `director` test scenario. `js/editor.js` is dormant (not imported). Rapier is lazy-loaded (2.2 MB) — an empty boot never pays for it; it loads during the boot preloader (menu buttons stay disabled until it's warm).

**Physics (`physics.js`) — determinism is the product.** Fixed 60 Hz timestep + accumulator; rendering interpolates between states and never feeds back. Zero `Math.random()` in sim code; world creation order is the backbone: ground → arena walls → props (array order) → cars (array order). Chassis colliders are convex hulls straight from each slab's `userData.pt` (+ big boxes/cylinders), rescaled to a per-category target mass (`CAT_PHYS`, scaled by footprint, × per-car `mass` multiplier) with a no-contact ballast collider that lowers the COM. Wheels come from `userData.wheel` tags (`parts.js` `wheel()`): duals dedupe to the outer wheel, inline bikes get invisible outriggers, tracked vehicles (dozer/excavator/roller/tram) get 4 virtual wheels at bbox corners. Raycast vehicle controller per car; front-cluster wheels steer, all wheels drive. Per-car tuning multipliers (`mass`/`grip`/`rest`/`crumple`) default to exactly 1.0 (or `rest: null` → 0.12) so legacy scenarios hash bit-identically. Launch phases in `stepOnce`: hold-with-brake until `delay`, then a `setLinvel` launch impulse (or none for `rolling` starts), full brake after `brake` seconds. Scenario shape: `{ world: {gravity, arena, walls}, props: [{kind,x,z,heading}], cars: [...] }`. "Perfect reset" = rebuild world + vehicles from the scenario (30 ms for 10 cars). **Incremental editing**: `replaceCar/appendCar/removeCarAt/setCarPose` (+ prop equivalents) touch only one object for paused-scene responsiveness — they do NOT promise a deterministic world, which is why the editor flags `dirty` and Play/Reset/Step always start runs from a full `build()`. **Damage model (crash-quality pass)**: `CAT_PHYS` carries per-category `sus` (suspension k/c/relax/travel×), `aDamp` (tall vehicles wallow, not snap-roll), `crumpleK` (deform softness — trucks are stiff steel) and `wheelTough`. Impacts charge nearby wheels; past `WHEEL_BENT_AT` a wheel steers crooked (bent offset + visible camber lean, grip cut), past `WHEEL_DETACH_AT` it tears off into a real dynamic debris body inheriting `v + ω×r` plus axle spin (visual wheel meshes re-parent to a debris node; `hashState` folds debris in; `_dropDebrisOf` cleans up on `replaceCar`/`removeCarAt`; `disposeSim` covers the rest). Resting contact at speed emits `onScrape`; deform's glass events surface via `onGlass`; wheel loss fires `onDetach`. All four hooks (`onImpact/onScrape/onGlass/onDetach`) run inside `stepOnce` and must stay render-side only — the one sanctioned feedback is `sim.speed` (wall-clock pacing; step math is untouched).

**Deformation (`deform.js`).** Geometry is non-indexed, so vertices weld by position (half-mm quantization) and displacement applies per weld group — faces never tear. Depth scales with contact Δv (impulse/mass, threshold 0.9 m/s), radial falloff + position-hash grain, accumulated crush clamped per group. Visual only — colliders stay rigid. Mesh↔chassis transforms are cached at build time (meshes never move relative to the wrap), so impacts convert world→mesh-local without touching render state. **v2 (crash-quality pass)**: per-group zone stiffness (nose/tail crumple deep, the cabin resists, the roof gives extra in rollovers), work hardening (already-crushed metal resists further hits), ridged crease noise + tangential shear so dents fold like beaten sheet metal instead of airbrushing, and two-stage glass — panes tagged `glass: true` through `M()` accumulate energy (direct impact-sphere overlap + inverse-square "frame shock" that pops windshields from bumper hits) through crack (cached crazed-material swap) to shatter (mesh hidden); crack/shatter records land in `state.events` and the sim drains them to `onGlass` in world space. Heavily crushed panels get a one-time paint scuff (material clone, render-only).

**Editor (`editor.js`).** The whole authoring UX: spawn flow (pick → Spawn → tap ground), object chips, selection (hover Box3Helper, selected outline), a custom transform gizmo (object-local X/Z move arrows + rotation ring + free-drag on the body, mouse and touch), the inspector (Spawn / Selected / Scene `<details>` sections in `index.html`), keyboard shortcuts, and the URL codec. URL: `scene=g~slow~arena~walls~env`, repeated `car=seed~type~paint~x~z~deg~v~th~st~mass~grip~rest~crumple~delay~brake~rolling` (first 9 fields = legacy v1 format, still parsed; seeds get `~` force-escaped) and `prop=kind~x~z~deg`. Old garage `?seed=&type=&paint=` links spawn that single car. Editing seed/type/paint of a car calls `sim.replaceCar(i)` — only that car rebuilds. Every edit sets `dirty`; Play (when starting), Step and Reset flush it with a full deterministic rebuild. Limits: 10 cars desktop / 6 mobile. Inputs are recorded per-tick streams (`makeStream`) so constant launch params can later become real driving input.

**Roads (`roads.js`).** Catmull-Rom spline roads (centripetal, so tight zig-zags never cusp): control points are world coordinates, `buildRoad(spec)` sweeps an asphalt ribbon + geometry lane markings (white dashes or double yellow), raised curbs, optional sidewalks and continental crosswalks near open ends, and returns `{ group, shapes }` — shapes is a flat list of curb box-collider recipes that physics attaches to ONE fixed body at the origin (`_addRoadRig`; the group is never transformed). Zero randomness: segment counts derive from arc length, so the same spec is bit-identical. Strip winding rule: horizontals face up when `o0<o1`; the mirrored (s=−1) side negates offsets AND flips winding — pass offsets ascending for s=+1 or faces render invisible from above. Spec `{ pts, w, loop, style }`, style bitfield: 1 = double yellow, 2 = sidewalks, 4 = crosswalks. World build order is ground → walls → roads → props → cars (legacy scenarios have no roads, so their hashes never moved). Editor: `roadDraw` button lays points on ground taps (Enter/button finishes, Esc cancels), roads are selectable (orange point handles drag individual points, dragging the asphalt moves the whole road, width/style in the inspector rebuild via `sim.replaceRoad(i)`), URL `road=w~loop~style~x,z~x,z~…`.

**World generators (`worldgen.js`).** `generateWorld(preset, seed, {maxProps, maxRoads})` (presets: suburb / city / highway) emits plain scenario content — `{ world: {arena, env}, roads, props }` — from the existing road + scenery pipelines, so generated worlds are ordinary editable objects and the URL captures the whole result (loading a shared link never runs the generator). One rng stream per `'w:'+preset+':'+seed`; placement walks the road spline via `roadCurve`/`getPointAt` with lateral "zones" (sidewalk edge / front yard / lot) chosen so nothing overlaps by construction. Props are pushed identity-first (anchors → lots → filler) and trimmed to `maxProps`, so smaller (mobile) budgets drop filler, never the scene's identity — but note that means the same seed gives a *smaller subset* on mobile, and that trim order is part of each generator's design. Gotcha learned: the city loop's left normal points OUT of the block for its point winding — outside-ring props use positive lateral offsets, the plaza uses centroid math. The highway preset was stretched for G1 (±130 m span, arena 290 — the old ±64 couldn't host a 10 s director approach, let alone oncoming traffic); its dressing offsets are arc-metric (`uM(m)`), so guardrail runs and cone tapers keep their spacing at any length. Editor: "World generator" chips + seed in the Scene section replace roads+props (cars are kept; if there are none, one spawns at the road start), then do a full `rebuildSim`. Editor caps were raised for generated worlds: 48 props / 6 roads desktop, 20 / 3 on small screens (~390 draw calls at full desktop budget — a merge/instancing pass is the next lever if caps ever grow).

**Props (`props.js`) & environments (`env.js`).** Props return `{group, bodies}` with *explicit* collider recipes (never parsed from geometry): ramp + barrier are fixed single-body hulls (node === the group), box stack is 6 dynamic crate bodies, pole is one dynamic body with `y` rest height (origin at its center). Dynamic prop nodes get re-parented to `sim.root` and sync straight from their bodies. Environments are strictly visual (nothing the sim touches): preset palettes + deterministic decoration groups over the shared 90 m ground disc; `proving` reproduces the tuned lighting baseline exactly, others may scale light intensities. Fog near/far live in `env.state` and are scaled by `env.setFogScale(fk)` from `fitCamera` — don't set `scene.fog` directly.

**Effects (`fx.js`).** Strictly render-side crash effects — reads sim state and hooks, never writes back (fx has its own rng; sim determinism is untouched by design). Pooled zero-alloc systems: three shader point clouds (additive sparks / normal-blend puffs for smoke-steam-dust-tire / additive fire) with viewport-scaled point sizes, instanced glass shards and paint-debris chips (tumble, ground bounce, shrink-out; chips take per-instance color from `car.paintHex`), a ground-decal ring buffer (skid ribbons traced from sliding wheels, scrape grinds, fluid-leak spots — deterministic y micro-offsets kill z-fighting), an impact flash PointLight, camera shake (applied around `renderer.render` only, then undone — orbit/freecam state never sees it), and procedural WebAudio (filtered-noise impacts/glass/detach + a persistent band-passed scrape loop; a one-shot cap keeps pileups from clipping into mush; the context needs a user gesture — `?crash=N` boots register a one-time pointerdown unlock). Per-frame emitters read rig damage: `frontDmg` staging steam → dark smoke → fire, plus fluid-drip decals; sliding or hard-braking wheels lay skids + tire smoke. `initFX(scene, {small})` halves pools on small screens; `attach(sim)` wires the four hooks; `update(dt, camera)` returns busy so render-on-demand still sleeps when idle (a burning wreck deliberately stays busy).

**Incident engine (`director.js`) — G1.** `generateScene(seed, d)` deals a complete deterministic round: separately derived rng streams per facet (`'scn:'+seed+':topo'/'incident'/'cast'/'dress'`, same convention as vehicles). Topologies: a bespoke signalized **intersection** (four road stubs + a 13 m `asphalt_patch` scenery infill 2 mm under the ribbons, lanes crossing straight through) plus **suburb / city / highway** built from `generateWorld` (lanes are extracted per direction at ±w/4 off each road spline; loops unroll into an open 85 % window). Eight templates (redlight · chain · blowout · drowsy · overspeed · pullout · police · brakefail), each returning cars with per-tick `drive` specs (polyline + cruise speed + sparse `cmds` timeline: `{t, v?, bias?, off?, noBrake?, brakeMax?}`) and optional template props (overspeed's corner "catchers"); everything goes wrong at exactly tick 600. Hard-won invariants encoded in the code comments: **nothing may collide before tick 600** (conflict scrub staggers crossing arrivals, then a spawn-overlap net runs BOTH before and after the scrub since the scrub moves cars); head-on templates (`needsOpp`) only deal where opposite-lane geometry exists — the oncoming victim's arrival window accounts for COMBINED closing speed (a victim reaching the meet point at T+n passes the aggressor at ~T+n/2, and the centreline crossing takes ~40–70 ticks, so windows sit at ~T+70–120); swerve cmds that must leave the lane need `off: true` or pure pursuit cancels the bias; heavies are barred from fast bendy plans (`v²·κ > 2.4` — a semi crabs across the centreline pre-incident); dressing is scrubbed off lane corridors, out of big-prop footprints (a tree standing inside a house collider gets ejected and falls across the road), and away from car spawns (footprint-aware radii). Difficulty drives the tell loudness (`tellK`), cast size, arrival-window spread, near-miss band (d ≥ 5), decoys (d ≥ 6), multi-incident (d ≥ 7), and template gating (`minD` — overspeed is d ≥ 5 by design: "does it actually crash?" is its market). `drawDifficulty(rng)` implements the campaign rarity curve.

**Pre-sim recorder (`recorder.js`) — G1.** `recordScene(R, scenario, catOf, {chunk, onProgress})` runs the scenario headlessly to rest and returns `{events, summary, restTick, ticks, hash, tracks}` — the event log the whole betting game settles against. Events (fixed-order, read-only scans inside the step loop): `hit` (deduped per car↔object pair per 20 ticks, with Δv + world position + attribution via the sim's `colToObj`), `glass`, `wheel`, `flip` (45 ticks inverted), `fire`, `escape`, `prop` (dynamic prop moved/toppled vs its tick-50 settled home), `close` (near-miss shave), `offroad` (left the asphalt at speed — nearest-road-sample test that ignores driving off the road END). Rest detection only counts INVOLVED bodies (touched cars, debris, disturbed props) + still-moving free traffic, so an untouched cruiser can't hold the scene open. The FNV event hash is the determinism fingerprint (bit-identical node↔browser). `chunk > 0` yields to the event loop so the round loading bar animates.

**Driver controller (in `physics.js`).** Cars with `spec.drive` get closed-loop pure pursuit along their polyline instead of a `makeStream` launch: arithmetic-only (`+,-,*,/,sqrt,min,max,abs` — no transcendentals, so bit-deterministic across JS engines), anticipatory `v² = 2as` braking into path ends, `end: 'stop'|'coast'`, sparse cmd timeline, post-crash "shock" (damage > 6 → brake and hold), and **ACC car-following for ambient cars** (`drive.acc = tick bound`): path-probe detection (goal point + far point + midpoint — a straight-ahead cone misses leaders around bends), same-direction gate (dot > 0.2 — reacting to crossing traffic shifts junction timing and breaks the conflict scrub), caps to the leader's TRUE velocity projected on own heading (a crabbing truck's odometer lies), full attention only before the incident tick (after it, panic range only — late brakers still pile in). Driven cars get real brake authority (`brakeK` 0.055 vs the legacy 0.02, which keeps old scenario hashes frozen). `sim.stopAt` freezes on an exact tick for the round's incident freeze; `colToObj` maps collider handles → `{kind: ground|wall|road|prop|debris}` for impact attribution.

**Conventions that everything relies on:**
- Vehicle space: forward = **+X**, ground = **y 0**, width along z. Builders return a `THREE.Group`; `buildVehicle()` recenters it on x/z, so builders never need to center themselves.
- Determinism: same seed+type+paint ⇒ identical vehicle. Type choice, build randomness, and naming each use a *separately derived* rng (`makeRng('t:'+seed)`, `'b:'+seed+':'+typeId`, `'n:'+...`) so forcing a type doesn't shift the build's random stream. Never call `Math.random()` inside builders — always the passed `r`.

**The geometry kit (`lib.js`)** is what produces the look; prefer it over raw boxes:
- `slab(mat, {x0,x1,y0,y1,w,wT,nose,tail,noseB,tailB,shiftT})` — a frustum defined by two rectangles (chamfered box). The returned mesh carries `userData.pt`, the parameter record.
- `faceQuad(pt, 'front'|'rear'|'left'|'right')` + `subQuad` + `quadPrism`/`panesOnQuad` — place thin plates *flush on any face of a slab, including sloped ones*. All windows, headlights, taillights, and grilles work this way (`P.headlightsOn(g, M, pt)` etc.). Caveat: `faceQuad` assumes slab-shaped `pt`; on `wedge()` (plan-tapered) faces the panes skew — use round `cyl` lamps there instead (see buggy/lemans).
- `hexa(bottom4, top4)` is the underlying 8-corner solid; winding/order is exact — copy an existing call rather than deriving anew.
- All geometry is non-indexed + `computeVertexNormals()` for flat shading; materials come from the per-build `matFactory()` (`M(hex, opts)`) which dedupes by parameter key.

**Family builders (`families.js`)** — `car()`, `truckFront()`+`chassis()`, `van()`, `bus()` — are config-driven; most of the 92 registry entries are just parameter sets. Only genuinely novel shapes (F1, dragster, excavator, tram, bikes…) get bespoke builders in `vehicles.js`.

**Registry (`vehicles.js` `REG`)**: `{ id, label, cat, build(r, M, ctx) }`. `ctx.paint` is the user's paint override and must win over any `paintHex` default (`bodyHex: ctx.paint || ...`). To add a vehicle: add a REG entry (UI dropdown, counts, and smoke test pick it up automatically), optionally add a flavor-name entry in `names.js` keyed by id.

**`main.js`** owns the game shell: renderer (ACES tone mapping, PMREM RoomEnvironment, shadow frustum refit per scene size), camera fit/tween, the showroom builder (flow layout: build → rotate to face +Z → measure bbox → wrap rows of ≤100 m; sections back→front are scenery → props/roads → vehicles), menu/pause/fullscreen/orientation flow, settings (env preset + Low/High quality: DPR 1 + shadows off vs full), Esc = pause, and boot (`?smoke`, `?simtest`, `?sheet`, `?play=1` = straight into the showroom without fullscreen). **Cameras**: Orbit (default) and **freecam** — pause-menu Camera chips, HUD 🎥 button, or `C`. Freecam is a true fly cam: pointer-lock mouse look + WASD (Space/Ctrl or E/Q vertical, Shift ×3, wheel = speed 5–140) on desktop; fixed left joystick + right-half look-drag + ▲▼ altitude buttons on touch (`#flyui`, shown via `body.ingame.flycam` + coarse pointer). Velocity is lerp-smoothed; `flyUpdate` returns false at rest so render-on-demand still sleeps. Critical: never run `controls.update()` while freecam owns the camera — OrbitControls re-aims at its target even when `enabled=false`. **Skybox** (`env.js` `SKIES`/`makeSky`): per-preset camera-following gradient dome + sun/moon disc + deterministic stars/cloud blobs, all `fog:false, toneMapped:false`; `scene.fog` is tinted to the sky horizon (the flat `bg` era is over; the tuned *lighting* values are unchanged) and `env.syncSky(camera.position)` runs every frame in the loop. **`editor.js`** (dormant) still holds the whole editor era: spawn flow, gizmo, inspector, URL codec, worldgen UI — see "Current app shell" above for what re-wiring needs. **Crash Test mode** (menu 💥 / `?crash=N`): `CRASH_SCENES` holds 8 seeded scene makers (`makeRng('crash:'+idx+':'+run)` — Next advances the scene AND rerolls the cast, Replay repeats the exact crash via perfect reset); `startCrash` lazy-loads physics + fx; `hookCrashSim` chains cinematics onto `onImpact` (auto slow-mo `sim.speed = 0.28` for 1.2 s on a ≥6.5 Δv hit, plus ONE `fitCamera` push-in per run on the first ≥5 Δv hit — both skipped while freecam owns the camera); a gentle auto-follow lerps `controls.target` to the wreck centroid in orbit mode; the damage HUD ticks every 250 ms and puts the sim to sleep once every body (debris included) rests, so render-on-demand can idle again.

## Crash Bet — the game (G1–G5)

**Status: plan done, implementation NOT started. Do not begin a phase until the user explicitly says go. Never push unless explicitly told.** Handoff protocol for a fresh session: read this section end to end, run the gates (`?smoke=1`, `?simtest=1`, node harness), find the first phase not marked **done** under Progress, build it to its acceptance criteria, update Progress, stop. This section is the product spec — treat it as the source of truth over any older description.

**The pitch.** A betting game about *reading physics*. Every round deals a seeded, procedurally generated traffic scene that is secretly ten seconds from disaster. The player watches it play, bets fake money on what the chaos will do — which car crashes, whether the hydrant pops, who flips, what survives — then time resumes and the deterministic sim settles every bet with zero ambiguity. One bankroll, $100 to start, skill = scene-reading. Vehicles and property only — no people, no gore, ever.

**One round, beat by beat.**
1. **Deal.** Campaign draws difficulty `d` 1–10 with geometric rarity (`P(d) ∝ 0.62^(d−1)` — level 1 ≈ 38 %, level 10 ≈ 0.4 %; the UI badges it, e.g. "LV 7 — top 3 % of scenes") and a scene seed from a hidden per-profile stream. The director assembles topology + cast + incident and **pre-simulates the entire outcome headlessly** (fixed steps to rest, ≤ 45 s sim-time after the incident) behind a themed loading beat ("syncing dashcams…"), producing a tick recording + event log. The odds engine NEVER reads the outcome — only settlement does.
2. **Watch (T−10 s → 0).** The scene plays live: normal-looking traffic with the tell hidden in plain sight (a van drifting in its lane, a light about to flip). Betting is open the whole time, from any camera.
3. **Lock (T = 0).** Time freezes on the incident tick. At d ≤ 7 the freeze holds — study the frozen scene, scrub the last 10 s, then press **BET** to resume. At d ≥ 8 there is no freeze: the slip must be in before T = 0, live. That pressure *is* the difficulty.
4. **Resolve (T = 0 → rest).** The incident fires; physics runs to rest. Auto slow-mo + camera push-in on the first big impact (already shipped in crash-test mode). Bet chips settle LIVE — "SUV crashes" flips green the instant its Δv threshold trips; "survives untouched" bets settle at rest.
5. **Payout.** Winnings count up, bankroll animates, a scene card summarizes (event timeline, best/worst pick), and every POV + free scrubbing unlocks for replay. NEXT deals a new scene. A settled seed can be re-watched forever but never re-bet.
6. **Bust.** $0 after settlement → ROCK BOTTOM card → restart at $100 (busts recorded in stats). Boot always resumes the last unfinished round — the seed re-deals deterministically and the slip draft is restored.

**Money rules.** Integer dollars only; min bet $1; max = bankroll; payout = ⌊stake × odds⌋. Singles ≈ x1.1–x50; a parlay multiplies its legs, capped x500. Odds v1 = incident-template outcome priors × geometry modifiers (distance to the incident path, mass class, fixed vs dynamic, shielding) × difficulty spread, minus house margin (~6 % baseline, easing to ~2 % at d ≥ 8 — hard reads pay nearly fair). Anti-cheese: campaign seeds are hidden until settlement; each seed settles winnings **once per profile** (ledger); custom or shared seeds always run as **Exhibition** (playable, no bankroll change).

**Markets (auto-generated per scene, all settled from the event log).**
- Headline: "A crash happens" / "No crash" — near-miss scenes exist at d ≥ 5, so this is a real market, not a freebie.
- Per vehicle: crashes (Δv ≥ threshold) · survives untouched · flips (roof/side rest) · loses a wheel · catches fire · first to crash · escapes the scene (exit zone).
- Per object — *anything targetable*: gets hit · destroyed/toppled · survives. Houses, poles, hydrants, mailboxes, trees, signs, benches…
- Scene-wide: over/unders (vehicles crashed, props destroyed), chain of ≥ 3, any car airborne, glass shatters.
- Template specials: "the runner gets T-boned", "the PIT sticks", "the jump clears the boxes".
- The slip: singles + one parlay group (v1), live combined odds, quick stakes $1 / $5 / $25 / Half / All-in.

**Cameras.** Each scene ships a loadout that difficulty prunes: dashcams on flagged cars (not every car has one), CCTV pole cam, traffic-signal cam, news-chopper orbit, witness tripod (auto-track + handheld shake). Freecam is always available and carries the **crosshair dot**: the center-screen raycast highlights whatever it rests on; click/tap opens that object's market card so you can bet on anything from anywhere. d 1–3 = full coverage; d 8+ can be a single grainy CCTV. After resolution, everything unlocks.

**Scene generation.** One master seed; separately derived streams per facet (`'scn:'+seed+':'+facet` — the same convention vehicles use) for topology, incident, cast, dressing, cameras — so forcing one facet never shifts another. Campaign seeds are 8-char base36 (≈ 2.8 × 10¹² distinct scenes); any custom string is a valid seed. Topology library grows to ~10 in G4: signalized intersection · suburb street · city block · highway + on-ramp · roundabout · bridge/causeway over water · mountain switchback with cliff guardrails · school zone · parking lot · rural tram crossing. Incident templates (8 in G1 → 20+ in G4), each declaring trigger choreography, per-difficulty parameter ranges, outcome priors, and its special markets: red-light runner, brake failure, tire blowout, drowsy drift, overspeed corner, chain rear-end, blind pullout, police chase (+ optional PIT), ramp jump, load spill, jackknife — plus d-gated **near-misses** (≥ 5), **decoys** — the car that looks doomed but recovers (≥ 6), and **multi-incident** scenes (≥ 7).

**What difficulty actually changes.** Incident subtlety (an obvious speeder vs a hairline brake-light flicker), cast size and similarity (five near-identical sedans at d 9), camera coverage, the freeze (removed at d ≥ 8), near-miss/decoy rates, and odds spread + margin. Never unfair randomness: every outcome is readable in the preview if you look at the right thing — d controls how hard "the right thing" is to spot.

**Persistence** (`localStorage["crashbet.profile.v1"]`, versioned JSON): bankroll, campaign cursor, current round `{seed, d, phase, slip draft}`, settled-seed ledger (ring buffer, 500), stats (rounds, hit-rate per market type, biggest win, longest streak, busts), settings. Reset = wipe after a confirm dialog.

## Gotchas learned the hard way

- **Color math must stay in sRGB**: pass `THREE.SRGBColorSpace` to `getHSL`/`setHSL` (see `shade`/`jitterColor` in `lib.js`). Linear-space HSL turns saturated reds salmon.
- Look-tuning baseline (don't casually change): exposure 1.18, key light 1.7, hemi 0.55, fill 0.45; glass `#1b2836` rough 0.32 env 0.85. Brighter key lights wash colors through ACES.
- `Mesh.position` is a read-only accessor — never `Object.assign(mesh, { position: ... })`; use `mesh.position.set(...)`.
- Tall truck beds (box/garbage/army/mixer) must pass `stacks: false` to `rigidTruck` or the exhaust stacks clip through the bed.
- Old vehicles must be disposed via `disposeGroup()` (in `lib.js`) before being replaced — `generate()` already does this.
- `run.bat`'s port cleanup uses `findstr /c:` deliberately: a bare space in a `findstr` pattern means OR and will match (and kill) far too much.
- Physics wheel discovery keys off `userData.wheel` set inside `parts.js` `wheel()` — a bespoke builder that rolls its own wheel cylinders won't get suspension (it'll fall back to virtual wheels). Prefer `P.wheel`/`P.axle`.
- The crash sim's visual wheel groups get re-parented (`wrap.attach`) at rig build so the controller can drive them; plain `buildVehicle()` builds (smoke test, contact sheet, .glb export of an unspawned car) are untouched.
- Prop bodies whose `node` IS the prop group must treat the node-local pose as identity in `_addPropRig` — reading `node.position` after the group was placed double-applies the prop offset (learned the hard way: ramp colliders 2× off).
- The embedded Browser pane freezes CSS transitions when `document.hidden` — a "wrong" `getBoundingClientRect` may just be a transition frozen mid-flight; disable the transition before measuring.
- Rapier objects need explicit cleanup: `CrashSim.dispose()` frees the world, event queue and controllers — never drop a sim without calling it (wasm memory leaks silently). Torn-off wheels are extra bodies + nodes — `destroyCrashSim`/`disposeSim` already cover them, but any new debris-spawning path must register with `this.debris` so cleanup and `hashState` see it.
- The `onImpact/onScrape/onGlass/onDetach` hooks fire *inside* `stepOnce`. They are render-side only — the sole sanctioned write-back is `sim.speed` (wall-clock pacing). Anything else (spawning a body, nudging a transform, reading `Math.random()`) breaks determinism and the `carnage` simtest will catch it. fx.js has its own `makeRng('fx')` for exactly this reason.
- `?crash=N` boots with no user gesture, so WebAudio starts suspended and stays silent until the first pointerdown (a one-time unlock listener handles it) — don't chase "no crash sound in headless" as a bug; it's the browser autoplay policy.
- The embedded Browser pane on this machine reports `document.hidden` and never fires `requestAnimationFrame`, so the game loop is frozen in that tab. Drive it with `__app.pump(now)` (synthetic clock) for any headless check — do NOT conclude the sim/camera/HUD is broken just because nothing moves on its own.
- `.claude/launch.json` uses `autoPort: true` (port 5183 is often held by another chat's server) — the preview picks a free port and prints it; don't hardcode 5183 in verification URLs, read the port from `preview_start`'s result. `run.bat` still pins 5183 for manual use.
